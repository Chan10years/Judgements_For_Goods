## 5. 阶段 8 建议任务清单

### 任务 1：设计采集配置结构

目标：

```text
定义采集开关、数据源类型、seed URL 路径、限速、超时、重试、输出路径和安全选项。
```

验收标准：

1. 配置可读。
2. 配置可审查。
3. 配置错误有明确提示。
4. 本地 JSON 和手动导入可作为兜底。

### 任务 2：设计 seed URL 读取流程

目标：

```text
读取客户提供的公开 URL 列表，校验格式，去重，并输出 seed_urls.json。
```

验收标准：

1. 有效 URL 被保留。
2. 重复 URL 被去重。
3. 无效 URL 被记录。
4. URL 来源可追踪。

### 任务 3：设计 robots 检查流程

目标：

```text
在抓取前检查目标站点 robots 规则，记录允许、拒绝和跳过原因。
```

验收标准：

1. robots 限制的 URL 被跳过。
2. 跳过原因写入报告。
3. 不绕过目标站点访问规则。

### 任务 4：设计 fetcher 模块

目标：

```text
按配置限速请求公开页面或客户授权数据源，记录 HTTP 状态、错误原因和重试情况。
```

验收标准：

1. 限速可配置。
2. 超时可配置。
3. HTTP 错误可记录。
4. 请求失败不影响兜底流程。

### 任务 5：设计 parser 模块

目标：

```text
从原始页面或授权数据源响应中提取商品标题、价格、店铺、链接、图片、参数文本、服务文本和原始文本。
```

验收标准：

1. 解析成功的商品进入后续流程。
2. 解析失败有原因。
3. 字段缺失有风险提示。
4. 原始来源可追踪。

### 任务 6：设计 normalizer 模块

目标：

```text
将不同来源商品统一成标准商品字段。
```

标准字段：

1. `platform`
2. `title`
3. `price`
4. `shop`
5. `url`
6. `image_url`
7. `specs_text`
8. `service_text`
9. `raw_text`
10. `source`
11. `evidence`
12. `manual_review_required`

验收标准：

1. 缺失字段能安全处理。
2. 价格尽量转成可比较格式。
3. 不能确认的字段标记为需人工复核。
4. 输出可被评分模块使用。

### 任务 7：设计 deduper 模块

目标：

```text
按 URL、标题、店铺和规格摘要对商品去重。
```

验收标准：

1. 重复商品能合并或跳过。
2. 去重原因可追踪。
3. 被合并记录有来源说明。

### 任务 8：设计 pipeline 模块

目标：

```text
串联配置、seed URL、robots 检查、fetcher、parser、normalizer、deduper、报告和日志。
```

验收标准：

1. 流程可单步检查。
2. 任一步失败有日志和报告。
3. 外部采集失败时可切回本地 JSON 或手动导入。
4. 不破坏阶段 1-7 已有主流程。

### 任务 9：设计 logging/report 模块

目标：

```text
生成 crawler_report.json 和 crawler_run.log。
```

报告必须记录：

1. 成功数量。
2. 失败数量。
3. 跳过数量。
4. robots 限制。
5. HTTP 错误。
6. 解析失败。
7. 字段缺失。
8. 去重结果。
9. 人工复核项。

验收标准：

1. 客户能看懂失败原因。
2. 开发者能根据日志定位问题。
3. 报告能支持阶段验收。

## 8. 客户交付验收重点

客户或客户指定验收人员重点检查：

1. 端到端流程可运行。
2. 商品数据采集可配置。
3. 采集结果可追踪。
4. 失败有日志和报告。
5. Word 输出可打开。
6. 推荐依据和风险提示清楚。
7. 无法确认的参数标记“需人工复核”。
8. 不把规则推荐结果称为最终采购结论。

## 8.1 阶段 8.1 验收口径

阶段 8 当前验收对象是采集底座 v1，而不是任意真实平台适配器。

验收方式：

1. 默认配置必须保守：尊重 robots、超时不低于 10 秒、请求间隔不低于 2 秒、单次 URL 不超过 20。
2. 默认 seed 文件为空，不预置淘宝、京东、1688、得物等复杂平台 URL。
3. 通过 mock pipeline 验证 raw、products、report、log 能生成。
4. 通过主流程回退验证外部采集为空或无效时仍可使用本地样例商品完成 Word 输出。
5. 真实客户 URL 采集属于后续客户数据源适配和授权验证范围，不承诺万能平台采集。
