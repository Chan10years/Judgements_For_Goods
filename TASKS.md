3. 阶段 1：Word 闭环任务
3.1 阶段目标
实现最小 Word 输入输出闭环：
上传 docx
↓
解析技术参数表
↓
页面展示解析结果
↓
生成模拟投标人响应值
↓
写回 docx
↓
下载 output.docx
3.2 任务清单
任务 1：创建基础项目结构
需要创建：
app.py
requirements.txt
src/
  doc_parser.py
  doc_writer.py
data/
outputs/
验收标准：
1.	项目结构清楚。
2.	app.py 可以作为入口。
3.	src 文件夹用于放核心模块。
4.	data 文件夹用于放样例数据。
5.	outputs 文件夹用于放输出文档。
任务 2：准备样例 Word
需要将样例 Word 放入项目中：
data/办公屏风01模板样例.docx
验收标准：
1.	样例 Word 存在。
2.	文件可以正常打开。
3.	后续解析模块可以读取该文件。
任务 3：实现 Word 表格解析
目标：
读取 Word 中的技术参数表。
需要提取字段：
1.	序号
2.	技术参数名称
3.	单位
4.	项目需求值或表述
5.	投标人响应值
输出结构：
[
  {
    "index": "1",
    "name": "尺寸要求（长×宽×高）",
    "unit": "mm",
    "required_value": "可按项目调整，1600×750×750",
    "response_value": ""
  }
]
验收标准：
1.	能读取样例 Word。
2.	能找到技术参数表。
3.	能提取所有参数行。
4.	页面或终端能看到解析结果。
5.	技术参数名称和项目需求值不能丢失。
任务 4：实现页面上传
目标：
用最小页面支持上传 docx。
验收标准：
1.	页面能选择 docx 文件。
2.	上传后能调用解析模块。
3.	页面能展示解析出的参数表。
4.	出错时能显示明确错误信息。
任务 5：生成模拟响应值
目标：
先不接商品数据，直接给每一条参数生成模拟响应内容。
示例：
已根据候选商品信息响应该参数，具体参数后续由商品匹配模块补充。
或：
模拟响应：该项参数待商品数据匹配后自动生成。
验收标准：
1.	每一行参数都有 response_value。
2.	response_value 不为空。
3.	模拟内容能写回 Word。
任务 6：实现 Word 写回
目标：
将模拟响应值写入“投标人响应值”列。
验收标准：
1.	能生成 output.docx。
2.	output.docx 可以正常打开。
3.	原技术参数表还在。
4.	“投标人响应值”列被填充。
5.	原文档结构尽量保持。
任务 7：页面下载输出 Word
目标：
在页面上提供下载按钮。
验收标准：
1.	用户可以下载 output.docx。
2.	下载后的文档可以打开。
3.	文档中能看到填充后的响应值。
3.3 阶段 1 完成标准
阶段 1 完成需要同时满足：
1.	页面可以上传样例 Word。
2.	页面可以展示解析出的技术参数。
3.	系统可以生成模拟响应。
