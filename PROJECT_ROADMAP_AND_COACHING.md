# 采购商品推荐与 Word 自动填充系统总路线与协作指南

## 1. 项目真实目标

客户要的不是单点小工具，而是“采购 Word 到商品推荐 Word”的交付闭环。

理想终态：

```text
客户上传采购 Word
→ 系统解析采购指标
→ 系统在淘宝、京东或合规商品数据源自动寻源
→ 系统形成候选商品池
→ 系统筛选、排序并推荐商品
→ 系统把推荐数据和风险提示写回 Word
```

当前可交付版本是 V1 半自动采购推荐与 Word 交付系统。V1 通过人机协同完成采购推荐闭环，不承诺淘宝/京东全自动搜索。

## 2. 当前已完成能力

阶段 1 到阶段 13.1 已经完成：

1. Word 读取和技术参数表解析。
2. 商品 JSON 数据读取与字段标准化。
3. 客户候选商品 JSON/CSV 适配器。
4. 公开 URL 采集底座、robots 检查、限速请求、解析、报告和日志。
5. 人工复核清单、复核模板、复核校验和复核回填。
6. 规则评分排序、Top 推荐、匹配理由和风险提示。
7. Word 参数响应写回、Top 推荐汇总和交付报告摘要。
8. Streamlit 操作界面。
9. 阶段 13.1 真实公开 URL 和半真实样例验收。

这些能力证明系统可以完成 V1 的后半段自动化：候选商品进入系统后，系统能整理、复核、排序并输出 Word。它不能证明当前版本已经具备淘宝/京东全自动搜索能力。

## 3. V1 交付版范围

V1 名称：

```text
半自动采购推荐与 Word 交付系统
```

V1 目标：

1. 支持采购员上传或读取采购 Word。
2. 自动解析采购指标。
3. 辅助生成淘宝/京东搜索关键词和搜索入口。
4. 支持采购员导入候选商品 JSON/CSV。
5. 自动标准化商品字段。
6. 自动生成待复核项。
7. 支持人工复核回填。
8. 自动规则排序。
9. 自动生成 Top 推荐。
10. 自动输出 Word 报告。

V1 演示重点：

```text
采购员负责确认候选商品来源。
系统负责把候选商品变成可审计、可排序、可写回 Word 的推荐报告。
```

## 4. V2 自动寻源范围

V2 名称：

```text
淘宝京东自动寻源与采购推荐系统
```

V2 目标：

1. 根据 Word 指标自动生成搜索任务。
2. 自动搜索淘宝、京东或合规商品数据源。
3. 自动获取商品标题、价格、店铺、链接、图片链接、规格、服务说明和证据文本。
4. 自动形成候选商品池。
5. 处理图片链接、图片证据和必要的可追溯证据。
6. 接入 V1 的复核、排序、Top 推荐和 Word 输出流程。

V2 必须先确认：

1. 平台授权接口。
2. 合规第三方商品搜索 API。
3. 客户导出的商品数据。
4. 可稳定访问的公开商品页面。
5. 明确的数据合规边界。

## 5. 最后一天执行计划

最后一天优先做阶段 13.2B，应急交付 V1 MVP：

1. 上传采购 Word。
2. 解析并展示采购指标。
3. 上传候选商品 JSON/CSV。
4. 生成淘宝/京东搜索关键词和搜索入口。
5. 运行商品字段标准化。
6. 输出人工复核清单。
7. 执行复核回填。
8. 运行规则排序和 Top 推荐。
9. 下载推荐 Word。
10. 补齐客户交付说明文档。

不要在最后一天切入淘宝/京东全自动抓取、图片识别、登录态处理、代理池或平台逆向。

## 6. 不做事项

V1 阶段禁止：

1. 不做全自动淘宝/京东抓取。
2. 不绕登录、验证码、风控。
3. 不偷 Cookie。
4. 不使用代理池。
5. 不自动下单。
6. 不抓取个人信息。
7. 不承诺最终采购结论。
8. 不编造商品字段、图片证据或来源证据。

## 7. 阶段路线重排

```text
阶段13.2A：全局交付路线重写与客户目标校准
阶段13.2B：应急交付 MVP，Word 上传与候选商品导入闭环
阶段14：错误处理和可维护性
阶段15：交付包整理
阶段16：最终验收文档
V2：淘宝京东自动寻源与图片证据处理
```

阶段 13.2A 只做文档校准。阶段 13.2B 才实现应急交付 MVP。阶段 14 延后，不作为当前下一步。

## 8. 协作规则

每次继续开发前必须明确：

1. 当前阶段。
2. 当前任务。
3. 允许修改文件。
4. 禁止修改文件。
5. 输入和输出。
6. 验收标准。
7. 安全红线。
8. 失败兜底。

涉及淘宝、京东、图片证据、自动搜索、自动抓取的需求，默认归入 V2，除非明确只做 V1 的搜索关键词和搜索入口辅助。

## 9. 客户交付话术

建议统一表达：

```text
当前版本支持半自动采购推荐闭环。系统可以解析采购 Word，辅助生成淘宝/京东寻源入口，并在采购员确认候选商品后，自动完成字段整理、人工复核、规则排序、Top 推荐和 Word 报告输出。

淘宝/京东全自动搜索、商品详情抓取和图片证据处理属于后续 V2，需要合规数据源或授权接口。
```
