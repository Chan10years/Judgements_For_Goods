# 采购商品推荐与 Word 自动填充系统项目总控文档

## 1. 项目真实目标

客户真实目标是形成“采购 Word 到商品推荐 Word”的闭环：

```text
上传或读取采购 Word
→ 解析采购指标
→ 寻源并形成候选商品
→ 字段整理与人工复核
→ 规则排序与 Top 推荐
→ 输出推荐商品数据和 Word 报告
```

最终理想能力包含淘宝、京东等平台的自动寻源、商品详情解析、图片证据处理和候选商品池自动生成。

当前可交付版本重新定义为：

```text
V1：半自动采购推荐与 Word 交付系统
V2：淘宝京东自动寻源与采购推荐系统
```

V1 解决客户最急的交付问题：系统支持人机协同采购推荐，帮助采购员完成采购指标解析、候选商品整理、匹配排序、风险提示和 Word 报告生成。V1 不承诺全自动抓取淘宝、京东搜索结果。

## 2. 当前已完成能力

阶段 1 到阶段 13.1 已完成以下能力：

1. 读取采购技术规范 Word。
2. 解析技术参数表中的采购指标。
3. 读取和标准化候选商品 JSON 数据。
4. 支持客户手动 JSON/CSV 候选商品数据适配。
5. 支持公开 URL 采集底座、失败报告、日志和字段缺失记录。
6. 生成人工复核清单。
7. 生成客户复核填写模板并校验复核文件。
8. 将人工 approved 的商品回填为已复核候选商品。
9. 根据采购指标和候选商品字段进行规则排序。
10. 生成 Top 推荐商品、匹配理由和风险提示。
11. 将参数响应、Top 推荐和交付摘要写回 Word。
12. 提供 Streamlit 操作界面，触发采集、推荐、复核模板、复核校验和复核回填。
13. 阶段 13.1 已验证真实公开 URL 请求链路、失败原因记录、字段缺失进入复核、半真实样例闭环和 Word 输出。

上述能力不能被描述为淘宝/京东全自动搜索系统。当前系统已经具备 V1 后半段自动化能力，但还没有淘宝/京东自动搜索适配器、真实平台商品页解析策略和图片证据处理闭环。

## 3. V1 交付版范围

V1 交付版定位为半自动采购推荐与 Word 交付系统。

V1 支持：

1. 上传或读取采购 Word。
2. 解析采购指标。
3. 导入候选商品 JSON/CSV。
4. 商品字段标准化。
5. 人工复核清单。
6. 复核文件校验。
7. 复核回填。
8. 规则排序。
9. Top 推荐。
10. Word 报告输出。
11. Streamlit 操作界面。
12. 淘宝/京东搜索关键词和搜索入口辅助。

V1 的淘宝/京东搜索入口辅助只包含关键词生成、可点击搜索链接或供采购员使用的寻源入口，不代表系统自动抓取淘宝/京东搜索结果。

V1 交付口径：

```text
当前版本支持半自动采购推荐闭环。
采购员确认或导入候选商品后，系统自动完成字段整理、人工复核、规则排序、Top 推荐和 Word 报告输出。
淘宝/京东全自动搜索属于后续 V2，需要合规数据源或授权接口。
```

## 4. V2 自动寻源范围

V2 目标能力：

1. 根据 Word 指标自动生成搜索任务。
2. 自动搜索淘宝、京东或合规商品数据源。
3. 自动获取候选商品标题、价格、店铺、链接、图片链接、规格、服务说明和证据文本。
4. 自动处理商品图片链接、图片证据和必要的截图/证据记录。
5. 自动形成候选商品池。
6. 接入 V1 已完成的复核、排序、Top 推荐和 Word 输出流程。

V2 前提条件：

1. 平台授权接口。
2. 合规第三方商品搜索 API。
3. 客户导出的商品数据。
4. 可稳定访问的公开商品页面。
5. 明确的数据合规边界。

没有上述前提时，不应承诺淘宝/京东全自动寻源。

## 5. 客户交付主流程

V1 客户交付主流程如下：

```text
采购 Word
→ Word 指标解析
→ 淘宝/京东寻源关键词和搜索入口辅助
→ 采购员确认候选商品或上传候选商品 JSON/CSV
→ 商品字段标准化
→ 人工复核清单
→ 复核回填
→ 规则排序
→ Top 推荐
→ Word 报告输出
→ 客户验收
```

## 6. 最后一天执行计划

最后一天优先完成阶段 13.2B 应急交付 MVP：

1. 打通上传采购 Word。
2. 打通上传候选商品 JSON/CSV。
3. 展示解析出的采购指标。
4. 展示候选商品标准化和字段缺失结果。
5. 输出人工复核清单和复核回填结果。
6. 运行规则排序和 Top 推荐。
7. 下载推荐 Word。
8. 提供淘宝/京东搜索关键词和搜索入口辅助。
9. 补齐客户交付说明文档。

阶段 13.2B 不进入淘宝/京东全自动抓取，不做图片证据处理闭环，只把 V1 的半自动采购推荐闭环交付稳定。

## 7. 不做事项和安全红线

V1 不做：

1. 不做全自动淘宝/京东抓取。
2. 不绕登录、验证码、风控。
3. 不偷 Cookie。
4. 不使用代理池。
5. 不抓取个人信息。
6. 不自动下单。
7. 不把采集结果、复核结果、排序结果或推荐结果称为最终采购结论。
8. 不把未确认字段写成已确认。
9. 不编造价格、规格、服务、来源、图片或证据文本。

所有输出必须说明结果为采购辅助候选，需要人工复核。

## 9. 风险和兜底

风险 1：客户误以为当前版本已能自动搜索淘宝/京东。

兜底：所有全局文档、交付说明和演示口径统一说明“当前版本为 V1 半自动采购推荐闭环”。

风险 2：淘宝/京东页面受登录、验证码、风控或动态页面影响。

兜底：V1 不承诺全自动抓取；V2 必须依赖授权接口、合规 API、客户导出数据或稳定公开页面。

风险 3：候选商品字段缺失或证据不足。

兜底：进入人工复核清单，不把缺失字段当成已确认字段。

风险 4：推荐结果被误解为采购结论。

兜底：Word、JSON、报告和客户话术均明确“采购辅助候选，非最终采购结论”。

## 10. 客户交付话术

建议对客户说明：

```text
当前版本支持半自动采购推荐闭环：上传采购 Word 后，系统解析采购指标，并辅助生成淘宝/京东寻源关键词和搜索入口。采购员确认或导入候选商品后，系统自动完成字段整理、人工复核、规则排序、Top 推荐和 Word 报告输出。

淘宝/京东全自动搜索、商品详情抓取和图片证据处理属于 V2 能力，需要平台授权接口、合规第三方商品搜索 API、客户导出数据或可稳定访问的公开商品页面作为前提。
```
